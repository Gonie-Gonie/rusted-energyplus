# Supported Object Matrix

The active object coverage table currently lives in
`operations/supported-object-coverage.md`.

Future conformance releases should publish a locked supported object matrix that
separates:

- parsed
- validated
- typed
- reference-resolved
- graph-resolved
- planned
- initialized
- simulated
- output-compared
- trace-compared

An object being parsed is not the same as being simulated or conformance-tested.

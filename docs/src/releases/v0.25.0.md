# v0.25.0 Release Notes

v0.25.0 closes the Opaque No-Mass Heat Balance Generalization milestone.

This release keeps the promoted numerical conformance set unchanged, but it
hardens the heat-balance runtime so opaque no-mass surfaces can represent more
than one boundary pattern before the next declared physics expansion.

## Added

- adiabatic opaque surface boundary handling
- interzone `Surface`, `Zone`, and `Space` boundary target resolution
- explicit missing boundary target errors
- runtime tests for adiabatic, interzone, and missing-target behavior
- `heat-balance-generalization-smoke`
- v0.25 release verification

## Changed

- `SurfaceHeatBalanceState` now records outside boundary object names and
  resolved adjacent surface/zone IDs.
- final surface face temperatures re-apply the boundary rule after zone air
  state is advanced.
- existing v0.8/v0.9 heat-balance conformance gates are part of the v0.25
  smoke path.
- release evidence report defaults now target v0.25.0 paths.

## Claim Boundary

The v0.25 claim remains limited-conformance for declared no-mass heat-balance
cases and variables only:

- `heat_balance_nomass_001`
- `surface_temperature_nomass_001`
- `Zone Mean Air Temperature`
- `Surface Inside Face Temperature`
- `Surface Outside Face Temperature`

It does not claim general heat-balance compatibility, dynamic exterior
envelope compatibility, fenestration, solar, warmup, sizing, HVAC, plant,
meter, or broad ExampleFiles compatibility.

The numeric conformance evidence PDF still contains the promoted v0.8/v0.9/v0.22
numerical cases only.

## Scope

Included:

- `node-state` output request class.
- `air_side_node_diagnostic_001` manifest and IDF.
- baseline-only EnergyPlus ESO evidence for:
  - `ZONE ONE INLET`
  - `ZONE ONE AIR NODE`
  - `ZONE ONE RETURN`
- node variables:
  - `System Node Temperature`
  - `System Node Humidity Ratio`
  - `System Node Mass Flow Rate`
- warning-free EnergyPlus baseline gate.
- typed node registry and NodeList graph evidence inherited from v0.10.
- report skeleton rows with first, last, min, max, and nonzero counts.
- v0.11 release verification and package artifact generation.

Conformance claims included in this release are still limited to earlier
declared tolerance-gated cases:

- `heat_balance_nomass_001`: `Zone Mean Air Temperature`
- `surface_temperature_nomass_001`: zone MAT plus surface inside/outside face
  temperatures

## Not Included

v0.11.0 does not claim:

- node temperature, humidity ratio, mass flow, or setpoint numerical parity
- `System Node Setpoint Temperature` parity
- IdealLoads load parity
- thermostat control parity
- fan, coil, PTAC, CAV, or air-loop component parity
- sizing or autosizing parity
- availability-manager parity
- outdoor-air, economizer, ventilation, heat-recovery, or humidity-control parity
- meter or fuel accounting parity
- general ExampleFiles compatibility

`air_side_node_diagnostic_001` remains:

```text
comparison_class: diagnostic-only
conformance_claim: false
tolerance_policy: none
status: baseline-only
```

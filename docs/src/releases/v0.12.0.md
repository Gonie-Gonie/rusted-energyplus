## Scope

Included:

- EnergyPlus 26.1.0 source-function map for system-node registration, storage,
  update, and output registration paths.
- links from the output-variable and HVAC porting maps to the node source map.
- `NodeStateProjectionEvidencePolicy` in `ep_runtime`.
- node projection markdown/JSON fields:
  - `source_map`
  - `timestamp_rule`
  - `warmup_rule`
  - `sentinel_rule`
  - `excluded_variable`
- EnergyPlus `SensedNodeFlagValue` boundary represented as `None` for future
  setpoint sampling.
- v0.12 release verification that checks source files, docs, Rust policy
  markers, diagnostic projection artifacts, tests, and false-claim boundaries.
- release PDF/HTML/JSON evidence pack for the promoted v0.8/v0.9 numerical
  conformance cases, including accuracy and execution-time graphs.

Conformance claims included in this release are still limited to earlier
declared tolerance-gated cases:

- `heat_balance_nomass_001`: `Zone Mean Air Temperature`
- `surface_temperature_nomass_001`: zone MAT plus surface inside/outside face
  temperatures

## Not Included

v0.12.0 does not claim:

- node temperature, humidity ratio, mass flow, or setpoint numerical parity
- `System Node Setpoint Temperature` parity
- IdealLoads load parity
- thermostat control parity
- fan, coil, PTAC, CAV, or air-loop component parity
- sizing or autosizing parity
- availability-manager parity
- outdoor-air, economizer, ventilation, heat-recovery, or humidity-control parity
- meter or fuel accounting parity
- general ExampleFiles compatibility

`air_side_node_diagnostic_001` remains:

```text
comparison_class: diagnostic-only
conformance_claim: false
tolerance_policy: none
status: baseline-only for the EnergyPlus report skeleton
status: projected for the Rust node-state projection
algorithm_parity: false for the Rust node-state projection
```

The Rust node-state projection artifact additionally records:

```text
source_map: docs/src/porting-map/node-state-source-map.md
timestamp_rule: hour-ending hourly samples aligned to the run-period time axis
warmup_rule: EnergyPlus warmup samples are not represented in this diagnostic projection
excluded_variable: System Node Setpoint Temperature
```

## Next

Future node or IdealLoads numerical work must use the source map to port the
EnergyPlus node update routines, define timestamp and sentinel handling, add
Rust result artifacts, declare tolerances, and introduce a blocking compare
gate before any claim is promoted.

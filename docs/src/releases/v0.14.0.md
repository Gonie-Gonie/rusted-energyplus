The artifact contains `bin/eplus-rs.exe`, scripts, docs source, config
examples, minimal testcases, conformance cases, conformance suites, and the
plant source-map documentation. When generated before packaging, it also
contains the release numerical conformance evidence pack under

Publishing is handled by `.github/workflows/release.yml` on `v0.14.0` tag push.
`scripts/dev.cmd github-release` is a local manual fallback.

## Scope

Included:

- `plant-source-map.md` source-function map for plant manager, loop side,
  branch/component dispatch, plant utilities, and first pump/boiler/chiller
  output paths.
- v0.14 release verification that checks EnergyPlus source files, source-map
  markers, Rust plant graph markers, docs, smoke output, tests, false-claim
  boundaries, evidence packaging, and package contents.
- release PDF/HTML/JSON evidence pack for the promoted v0.8/v0.9 numerical
  conformance cases.

Conformance claims included in this release are still limited to earlier
declared tolerance-gated cases:

- `heat_balance_nomass_001`: `Zone Mean Air Temperature`
- `surface_temperature_nomass_001`: zone MAT plus surface inside/outside face
  temperatures

## Not Included

v0.14.0 does not claim:

- plant numerical conformance
- plant loop flow balancing or operation-scheme parity
- pump electricity, boiler load, chiller load, or condenser-loop parity
- plant node temperature, mass-flow, setpoint, or plant meter parity
- plant ExampleFiles compatibility
- HVAC, node, IdealLoads, sizing, autosizing, availability-manager, or general
  ExampleFiles compatibility beyond already promoted cases

The plant skeleton fixture remains:

```text
evidence_level: smoke
conformance_claim: false
tolerance_policy: none
status: typed graph only
source_map: docs/src/porting-map/plant-source-map.md
```

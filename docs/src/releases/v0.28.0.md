# v0.28.0 Release Notes

v0.28.0 enriches input object coverage metadata for the Support Coverage
Report.

This release does not add new numerical conformance. It adds first evidence
and support-boundary fields to every tracked input object so users can see not
only whether an object is typed or graph-only, but also where that support is
exercised and what it does not imply.

## Added

- `first_evidence` for every tracked input object
- `support_boundary` for every tracked input object
- generated mdBook object coverage columns for first evidence and boundary
- support coverage PDF/HTML/JSON/Markdown input tables with first evidence and
  boundary columns
- v0.28 release verification

## Changed

- release evidence defaults now target `0.28.0`
- support coverage JSON now includes
  `input_objects_with_first_evidence_count`

## Claim Boundary

The v0.28 report metadata is documentation infrastructure. It does not promote
new conformance, full EnergyPlus compatibility, HVAC numerical conformance,
plant numerical conformance, or meter conformance.

As of v0.28.0, the generated report covers:

- 20 tracked input objects
- 20 input objects with first evidence
- 45 tracked output variables from specs plus manifests
- 78 manifest output requests
- 4 tracked algorithm families

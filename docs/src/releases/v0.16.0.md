## Scope

Included:

- `run plant-state-projection` diagnostic command.
- `PlantStateProjectionEvidencePolicy` with source-map, timestamp, warmup, and
  sizing-period policy fields.
- `plant-loop-projection-smoke` gate for Rust projection artifacts.
- projected `plant-state` and `plant-equipment` Rust `ResultStore` rows for
  the v0.15 PlantLoadProfile fixture.
- release PDF/HTML/JSON evidence pack for the promoted v0.8/v0.9 numerical
  conformance cases.

Conformance claims included in this release are still limited to earlier
declared tolerance-gated cases:

- `heat_balance_nomass_001`: `Zone Mean Air Temperature`
- `surface_temperature_nomass_001`: zone MAT plus surface inside/outside face
  temperatures

## Not Included

v0.16.0 does not claim:

- plant numerical conformance
- plant loop flow balancing or operation-scheme parity
- pump electricity, district heating, load profile, boiler load, chiller load,
  condenser-loop, or plant node-state parity
- plant meter parity
- plant ExampleFiles compatibility
- HVAC, node, IdealLoads, sizing, autosizing, availability-manager, or general
  ExampleFiles compatibility beyond already promoted cases

The Rust plant projection remains:

```text
comparison_class: diagnostic-only
conformance_claim: false
algorithm_parity: false
tolerance_policy: none
status: projected
```

## Next

Future plant work should replace diagnostic projections with source-mapped
plant state fields driven by ported loop-side, component dispatch, flow
request, and equipment algorithms before any tolerance-gated plant comparison
is proposed.

# v0.24.0 Release Notes

v0.24.0 closes the Runtime State and Output Registry Hardening milestone.

This release does not add new numerical conformance cases. It strengthens the
runtime infrastructure needed before v0.25 expands the no-mass heat-balance
surface set.

## Added

- `RuntimeOutputRegistry`
- `RuntimeMeterRegistry`
- `RuntimeOutputRequest` and `RuntimeMeterRequest`
- explicit `OutputVariableUnavailable` and `MeterUnavailable` diagnostics
- duplicate output request, handle, and series diagnostics
- `ResultStoreProfile`
- `runtime-registry-smoke`
- v0.24 release verification

## Changed

- `build_execution_plan` now resolves output stage handles from the runtime
  output registry instead of using a placeholder output handle.
- `ResultStore` and output-related runtime primitives are split into
  `crates/ep_runtime/src/output.rs`.
- `model-plan-smoke` now verifies registry-backed output steps.
- release evidence report defaults now target v0.24.0 paths.

## Claim Boundary

The v0.24 claim is runtime-infrastructure only. It covers:

- registry construction for currently implemented runtime output variables
- explicit unavailable-output and unavailable-meter diagnostics
- result-store duplicate-handle and duplicate-series checks
- profile scaffolding for future report evidence

It does not claim new numerical conformance, meter conformance, general runtime
compatibility, HVAC compatibility, plant compatibility, or broad ExampleFiles
compatibility.

The numeric conformance evidence PDF remains limited to promoted v0.8/v0.9/v0.22
numerical cases. Static v0.23 EIO evidence remains represented by the static
gate artifacts and the conformance index.

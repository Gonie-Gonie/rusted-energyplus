# v0.27.0 Release Notes

v0.27.0 adds the user-facing Support Coverage Report milestone.

This release does not add new numerical conformance. It packages the current
supported input, output, and algorithm scope into oodocs-generated
PDF/HTML/JSON/Markdown artifacts so users can see exactly what is supported,
diagnostic-only, or graph-only.

## Added

- `support-coverage-report` release command
- `tools/reporting/support_coverage_report.py`
- oodocs/matplotlib PDF and HTML support coverage report
- machine-readable `support-coverage-report.json`
- user documentation for reading support coverage status
- v0.27 release verification

## Changed

- release evidence defaults now target `0.27.0`
- `check` now generates the support coverage report
- mdBook navigation exposes the Support Coverage Report page
- release report-format docs list support coverage artifacts beside numerical
  conformance and conformance-index evidence

## Claim Boundary

The v0.27 report is a coverage map. It does not create new conformance claims.

As of v0.27.0, the generated report covers:

- 20 tracked input objects
- 45 tracked output variables from specs plus manifests
- 78 manifest output requests
- 4 tracked algorithm families
- 13 conformance or diagnostic cases

It does not claim full EnergyPlus compatibility, broad ExampleFiles numerical
compatibility, HVAC numerical conformance, plant numerical conformance, meter
conformance, sizing, EMS, PythonPlugin, daylighting, fenestration, solar, or
warmup compatibility.

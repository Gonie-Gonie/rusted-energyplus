# v0.23.0 Release Notes

v0.23.0 closes the Static Model Evidence Expansion milestone.

This release promotes one official ExampleFile static model case:

- `official_1zone_static_model_001`

The case uses EnergyPlus 26.1.0 `ExampleFiles/1ZoneUncontrolled.idf` and
compares Rust typed-model interpretation against EnergyPlus EIO rows for
declared static fields.

## Added

- `conformance static-model-report`
- `compare-static-model-conformance`
- `official_1zone_static_model_001`
- staged `Output:Surfaces:List,Details` injection for EIO surface-detail rows
- v0.23 coverage in the conformance index PDF/HTML/JSON/Markdown report

## Claim Boundary

The v0.23 claim is static-evidence only. It covers declared static EIO rows for:

- heat-transfer surface class, net/gross area, azimuth, and tilt
- Construction CTF layer count and thermal conductance
- Material CTF Summary thickness, conductivity, density, specific heat, and
  thermal resistance
- OtherEquipment Internal Gains Nominal floor area, equipment level,
  equipment per floor area, and latent/radiant/lost/convected fractions

It does not claim dynamic heat-balance compatibility, HVAC compatibility,
plant compatibility, solar, fenestration, warmup, sizing, meter conformance,
or broad ExampleFiles compatibility.

The numeric conformance evidence PDF remains limited to the previously
promoted v0.8/v0.9/v0.22 numerical cases. Static EIO evidence is represented
by the static gate artifacts and the conformance index.

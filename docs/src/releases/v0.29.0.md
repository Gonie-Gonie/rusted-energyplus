# v0.29.0 Release Notes

v0.29.0 enriches output variable coverage metadata for the Support Coverage
Report.

This release does not add new numerical conformance. It makes the output
coverage report safer to read by resolving each variable's first evidence from
the strongest available evidence level, then showing observed levels, best
evidence cases, sources, frequencies, and support boundaries.

## Added

- output variable support boundaries in `specs/variable_coverage.toml`
- generated mdBook variable coverage columns for first evidence and boundary
- support coverage JSON fields for output variable first-evidence and boundary
  completeness
- output table columns for observed levels, best evidence cases, and support
  boundary
- v0.29 release verification

## Changed

- output variable first evidence now prefers promoted conformance evidence over
  earlier diagnostic or smoke appearances with the same variable name
- release evidence defaults now target `0.29.0`

## Claim Boundary

The v0.29 metadata is documentation infrastructure. It does not promote new
conformance, full EnergyPlus compatibility, HVAC numerical conformance, plant
numerical conformance, or meter conformance.

As of v0.29.0, the generated report covers:

- 20 tracked input objects
- 45 tracked output variables
- 45 output variables with support boundaries
- 78 manifest output requests
- 4 tracked algorithm families

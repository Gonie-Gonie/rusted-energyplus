# v0.30.0 Release Notes

v0.30.0 enriches algorithm coverage metadata for the Support Coverage Report.

This release does not add new numerical conformance. It adds first evidence and
support boundaries to tracked algorithm entries so users can distinguish
limited-conformance algorithms from diagnostic projections without reading the
developer source maps first.

## Added

- `first_evidence` and `support_boundary` metadata for all tracked algorithms
- generated mdBook algorithm ledger columns for first evidence and boundary
- support coverage JSON fields for algorithm first-evidence and boundary
  completeness
- support coverage algorithm table boundary column
- v0.30 release verification

## Changed

- release evidence defaults now target `0.30.0`

## Claim Boundary

The v0.30 metadata is documentation infrastructure. It does not promote new
conformance, full EnergyPlus compatibility, HVAC numerical conformance, plant
numerical conformance, or meter conformance.

As of v0.30.0, the generated report covers:

- 20 tracked input objects
- 45 tracked output variables
- 4 tracked algorithm families
- 4 algorithms with first evidence
- 4 algorithms with support boundaries

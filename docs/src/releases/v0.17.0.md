## Scope

Included:

- `manifest_v2` metadata in every tracked conformance case.
- `[scope]` feature/domain metadata in every tracked conformance case.
- output request v2 fields: `domain`, `level`, and optional per-output
  tolerances.
- meter request schema support.
- waiver schema support.
- `eplus-rs conformance validate-case-v2 <case.toml>`.
- `manifest-validate-all` dev gate.
- v0.17 release verification gate.

Conformance claims included in this release are still limited to earlier
declared tolerance-gated cases:

- `heat_balance_nomass_001`: `Zone Mean Air Temperature`
- `surface_temperature_nomass_001`: zone MAT plus surface inside/outside face
  temperatures

## Claim Boundary

v0.17.0 does not claim:

- a new numerical conformance case.
- ExampleFiles compatibility.
- HVAC, node, plant, meter, sizing, or autosizing conformance.
- broader heat-balance compatibility beyond the already promoted v0.8/v0.9
  cases.

The v2 gate is schema and evidence-contract infrastructure. It is required
before v0.18 output injection and official-file baseline expansion.

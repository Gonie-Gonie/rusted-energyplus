# v0.21.0 Release Notes

v0.21.0 closes the Source Map and Algorithm Ledger v1 milestone.

## Scope

- adds `source_map` links to `specs/algorithm_ledger.toml`
- updates generated algorithm ledger docs with a source-map column
- validates EnergyPlus 26.1.0 source files against the repo-local reference
  source tree
- validates Rust target files and function/type anchors
- validates first-case manifests, proof variables, conformance claim status,
  and blocking gates for promoted ledger entries
- adds `algorithm-ledger-check` and `v0.21-verify`
- packages `specs/` with the release zip

## Release Artifacts

- `dist/eplus-rs-v0.21.0-windows-x64.zip`
- `conformance-index-report.html`
- `conformance-index-report.pdf`
- `conformance-index-report.json`
- `numeric-conformance-evidence.html`
- `numeric-conformance-evidence.pdf`
- `numeric-conformance-evidence.json`

## Claim Boundary

This release does not add numerical conformance. It is a planning guard that
prevents algorithm-port claims without source-map, Rust target, case, variable,
report, tolerance, and blocking-gate evidence.

Promoted tolerance-gated numerical conformance remains limited to:

- `heat_balance_nomass_001`
- `surface_temperature_nomass_001`

## Next

v0.22 should expand time, weather, and schedule conformance only for explicitly
declared variables with tolerance policies and release gates.

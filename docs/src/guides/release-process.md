# Release Process

Release checks are launched through `scripts/dev.cmd`. Version-specific gates
remain available for completed milestones, while current planning should prefer
spec-led verification and generated evidence.

The local release package command is:

```powershell
.\scripts\dev.cmd package -Version 0.24.0
```

The numerical evidence document command is:

```powershell
.\scripts\dev.cmd conformance-evidence-report -Version 0.24.0
.\scripts\dev.cmd conformance-index-report -Version 0.24.0
```

Evidence and index documents are generated by Python using oodocs and
matplotlib, with PowerShell acting only as the entry point.

Generated release evidence is written under `.runtime/release-evidence/vX.Y.Z`
for local inspection and uploaded as separate GitHub Release assets. The binary
zip is not the evidence store. Reports prove claims; Markdown release notes
describe them.

# v0.3.0 Readiness

Status: in progress.

## Release Definition

v0.3.0 introduces the first runtime-ready compile artifact shape:

```text
TypedModel -> SimulationModel -> ModelGraph -> ExecutionPlan
```

Supported command:

```powershell
eplus-rs model plan <input.epJSON>
```

## Completion Checklist

| Requirement | Evidence |
|---|---|
| SimulationModel | `ep_model::SimulationModel` wraps typed data and graph relations |
| Zone-Surface graph | `ModelGraph.zone_surfaces` records `ZoneId -> SurfaceId` edges |
| Construction-Material graph | `ModelGraph.construction_materials` records first layer material edges |
| ExecutionPlan skeleton | `ep_runtime::build_execution_plan` emits environment, zone, and output stages |
| plan CLI | `eplus-rs model plan` prints graph and execution-plan summary |
| smoke test | `scripts\model-plan-smoke.ps1` verifies fixture graph and plan counts |

## Verification Command

```powershell
.\scripts\model-plan-smoke.cmd
.\scripts\check.cmd
```

## Boundary

This is not a persisted binary artifact yet. The current artifact is a
deterministic in-memory structure and CLI-readable summary. v0.6 builds on this
by executing the plan into a `ResultStore`.

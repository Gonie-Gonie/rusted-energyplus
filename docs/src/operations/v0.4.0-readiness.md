# v0.4.0 Readiness

Status: ready when `.\scripts\dev.cmd v0.4-verify` passes.

## Release Definition

v0.4.0 locks the first time, weather, and schedule evidence surface:

```text
RunPeriod + hourly time axis + EPW fields + Schedule:Constant + Schedule:Compact subset
```

Supported comparison commands:

```powershell
eplus-rs compare schedule-value <input.epJSON> <eplusout.eso>
eplus-rs compare weather-fields <weather.epw> <eplusout.eso>
```

## Completion Checklist

| Requirement | Evidence |
|---|---|
| `RunPeriod` intake | `ep_compiler` parses date ranges and `ep_runtime` expands inclusive hourly axes |
| weather field parser | `ep_runtime::parse_epw_records` reads dry-bulb, dew point, RH, pressure, radiation, and wind fields |
| weather field comparison | `compare-weather-smoke.ps1` compares 6 selected EPW fields with EnergyPlus ESO |
| `Schedule:Constant` comparison | `compare-schedule-smoke.ps1` compares `Schedule Value` with EnergyPlus ESO |
| `Schedule:Compact` subset | `schedule-compact-smoke.ps1` verifies all-days `Until` segment intake |
| manifest evidence | `schedule_constant_001` and `weather_fields_001` case manifests declare outputs |
| report skeletons | `conformance-report-smoke.ps1` writes baseline-only schedule and weather reports |
| false-claim boundary | manifests use `comparison_class = "smoke"` and `conformance_claim = false` |

## Verification Command

```powershell
.\scripts\dev.cmd v0.4-verify
```

## Boundary

This milestone does not support full `Schedule:Compact` day-type/calendar
expansion, solar/radiation ESO field comparison, heat-balance conformance, HVAC,
or plant simulation. The report artifacts are baseline-only smoke evidence
until Rust result artifacts, declared tolerances, and blocking gates are added.

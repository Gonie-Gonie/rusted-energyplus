# v0.10.0 Plan

Purpose: promote thermostat, zone equipment, and IdealLoads intake from
raw-only coverage to a typed model and graph foundation.

## Blocking Smoke Case

`ideal_loads_thermostat_001` is the v0.10 typed-graph gate.

It covers:

- one `Zone`
- one `ThermostatSetpoint:DualSetpoint`
- one `ZoneControl:Thermostat`
- one `ZoneHVAC:EquipmentConnections`
- one `ZoneHVAC:EquipmentList`
- one `ZoneHVAC:IdealLoadsAirSystem`
- six opaque adiabatic `BuildingSurface:Detailed` surfaces
- hourly thermostat heating and cooling setpoint variables
- hourly IdealLoads total heating and cooling rate variables

The case manifest declares:

- `comparison_class = "smoke"`
- `conformance_claim = false`
- no tolerance rules
- `zone-state` thermostat output variables
- `hvac-state` IdealLoads output variables
- blocking smoke gate: `scripts/dev.cmd ideal-loads-thermostat-smoke`

## Typed Graph Contract

The v0.10 gate requires the generated epJSON to compile with:

- `thermostat_dual_setpoints: 1`
- `zone_thermostats: 1`
- `ideal_loads_air_systems: 1`
- `zone_equipment_lists: 1`
- `zone_equipment_connections: 1`

The execution plan must expose:

- `zone_thermostat_edges: 1`
- `thermostat_setpoint_edges: 1`
- `zone_ideal_loads_edges: 1`

This makes the control and equipment relationships inspectable before any
IdealLoads numerical solver is claimed.

## Not Claimed

This is not an IdealLoads load-conformance claim.

Not claimed in v0.10:

- IdealLoads heating or cooling rate parity
- thermostat control-loop numerical parity
- HVAC availability, sizing, economizer, heat recovery, ventilation, humidity,
  or fuel-use parity
- system-node mass flow, temperature, or humidity-ratio conformance
- plant, fan, coil, outdoor-air system, or air-loop behavior
- warmup, sizing-period, or unmet-load parity

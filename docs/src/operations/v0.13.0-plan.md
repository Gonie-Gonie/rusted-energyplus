# v0.13.0 Plan

Purpose: introduce the first typed PlantLoop graph skeleton before any plant
solver, node-flow, equipment, meter, or ExampleFiles numerical claim is
promoted.

This is not a plant numerical conformance claim.

## Evidence Targets

- `PlantLoop`, `Branch`, `BranchList`, `Connector:Splitter`,
  `Connector:Mixer`, and `ConnectorList` compile into typed model records.
- `Pump:ConstantSpeed`, `Boiler:HotWater`, and `Chiller:Electric:EIR` compile
  as the first plant equipment identity records.
- plant inlet/outlet nodes are registered in the shared node registry.
- `ExecutionPlan` exposes graph edges from loops to branch lists, branch lists
  to branches, connector lists to connectors, and branches to component records.
- `plant-loop-skeleton-smoke` gates the minimal epJSON fixture without using it
  as conformance evidence.
- `plant.md` records the source-map targets needed before future plant
  numerical work can be promoted.

## Release Gate

```powershell
.\scripts\dev.cmd v0.13-verify
```

## Boundary

Allowed in v0.13:

- typed intake for the listed plant loop and equipment objects
- reference resolution among loop, branch list, branch, connector list, and
  connector records
- graph edge summaries for the plant skeleton fixture
- smoke-level evidence that the typed graph is present
- packaging of the existing v0.8/v0.9 numerical conformance PDF/HTML/JSON
  evidence under `evidence/v0.13.0`

Not allowed in v0.13:

- plant loop flow balancing, operation schemes, pump head, boiler load, chiller
  load, condenser-loop, or common-pipe parity claims
- plant node temperature, humidity, mass-flow, setpoint, or meter numerical
  conformance claims
- plant ExampleFiles compatibility wording
- promotion of the plant skeleton fixture beyond smoke evidence

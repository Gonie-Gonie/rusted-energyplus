# v0.3.0 Plan

Purpose: input interpretation contract.

Evidence targets:

- object count comparison
- default handling comparison
- name/reference resolution report
- unsupported object classification
- IDD/schema/epJSON field mapping report

This milestone is about interpreting EnergyPlus input, not runtime heat-balance
compatibility.

`SimulationModel`, `ModelGraph`, and `ExecutionPlan` may exist as foundations,
but they are not v0.3 public compatibility evidence unless promoted with their
own cases, output variables, artifacts, and gates.

Decision for this reset: graph and execution-plan behavior stays foundation-only
in v0.3. Release evidence for graph/runtime/output infrastructure moves to
v0.6 or later, where it must have its own manifests, artifacts, reports, and
non-conformance boundary language.

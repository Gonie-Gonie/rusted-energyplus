# v0.2.0 Readiness

Status: in progress.

## Release Definition

v0.2.0 turns the v0.1 TypedModel preview into an explicit coverage and
diagnostics contract.

Supported command:

```powershell
eplus-rs model compile <input.epJSON>
```

## Completion Checklist

| Requirement | Evidence |
|---|---|
| typed model compile CLI coverage contract | compile output includes `coverage` with `typed` and `raw-only` statuses |
| enum/default/unit handling | typed fixture checks defaults and enum parsing |
| NameMap | `ep_model::NameMap` normalizes trim plus ASCII case folding |
| typed ID | surfaces, constructions, materials, schedules, and zones use typed IDs |
| missing reference diagnostics | `missing-reference.epJSON` smoke expects `MissingReference` |
| invalid enum diagnostics | `invalid-enum.epJSON` smoke expects `InvalidEnumValue` |
| oracle-generated epJSON smoke | typed smoke compiles `.runtime/oracle-smoke/26.1.0/convert/smoke.epJSON` |
| first EnergyPlus/Rust series comparison | `compare-schedule-smoke.ps1` compares `Schedule Value` from ESO |
| first weather comparison | `compare-weather-smoke.ps1` compares EPW dry-bulb values to EnergyPlus ESO |

## Verification Command

```powershell
.\scripts\typed-model-smoke.cmd
.\scripts\compare-schedule-smoke.cmd
.\scripts\compare-weather-smoke.cmd
.\scripts\check.cmd
```

## Boundary

v0.2.0 still does not build graph artifacts, execution plans, or simulation
state from the model. The schedule comparison is intentionally limited to
`Schedule:Constant` and weather dry-bulb values. These prove the oracle/Rust
series comparison loop before graph, zone heat balance, or HVAC comparisons are
introduced.
Unsupported object types are preserved and reported as `raw-only` rather than
rejected.

# v0.6.0 Plan

Purpose: lock output, trace, compare, and diagnostic report infrastructure
without claiming heat-balance or zone-temperature numerical conformance.

Evidence targets:

- native `ResultStore` diagnostic path from `run first-zone`
- diagnostic-only `compare zone-temperature` extraction with
  `comparison_class: diagnostic-only`, `conformance_claim: false`,
  `tolerance_policy: none`, and `status: extracted`
- generated `compare-summary.json` and `compare-report.md` for the
  zone-temperature diagnostic
- manifest-driven diagnostic report generation for
  `zone_temperature_diagnostic_001`
- compare regression artifacts:
  `trace.json`, `compare-summary.json`, `compare-report.md`, and
  `profile-summary.json`

Release gate:

```powershell
.\scripts\dev.cmd v0.6-verify
```

Decision for this reset: keep `run first-zone` available for developer
diagnostics in v0.6, but do not describe it as a public simulation capability.
The public release contract is the diagnostic artifact boundary, not a zone air
temperature pass. A later CLI namespace cleanup can move developer-only
commands without changing the v0.6 evidence claim.

This milestone does not introduce meter comparison, SQL comparison,
full output-variable patching, tolerance-gated zone-temperature reports,
surface heat-balance conformance, HVAC, plant, or whole-building simulation
compatibility.

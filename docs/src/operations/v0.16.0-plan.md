# v0.16.0 Plan

Purpose: reset versioning and evidence language before continuing Road to v1.0.

v0.16 is the Versioning and Evidence Cleanup milestone. The Rust plant-state
projection work added after v0.15 is retained as an additional diagnostic
artifact, but it is not the defining purpose of v0.16.

This is not a plant, HVAC, node, meter, sizing, or ExampleFiles numerical
conformance claim, and it is not a general numerical conformance claim.

## Evidence Targets

- `versioning-reset-v2.md` becomes the canonical roadmap reset.
- `legacy-milestones.md` classifies v0.1 through v0.15 as the Historical
  Pre-Alpha Evidence Series.
- `milestone-map.md` is rewritten around Road to v1.0 and long-term v1/v2/v3
  targets.
- `v1-scope.md`, `v2-full-compatibility.md`, and `v3-fast-successor.md` define
  the meaning and boundaries of the major version targets.
- `v0.17.0-plan.md` and `v0.18.0-plan.md` start the next infrastructure steps.
- README and release notes avoid reading v0.10 through v0.15 as broad feature
  completion.
- Release language includes a Claim Boundary section.
- The plant-state projection addendum stays gated as diagnostic-only plumbing
  with `algorithm_parity: false`.

## Release Gate

```powershell
.\scripts\dev.cmd v0.16-verify
```

## Boundary

Allowed in v0.16:

- versioning cleanup and roadmap documentation.
- explicit v0.1-v0.15 legacy interpretation.
- v1/v2/v3 target scope documents.
- Road to v1.0 plan entries.
- packaging of the promoted v0.8/v0.9 numerical evidence pack.
- diagnostic plant-state projection addendum as non-claim runtime plumbing.

Not allowed in v0.16:

- broad compatibility wording.
- promotion of v0.10 through v0.15 into implementation-complete claims.
- plant loop solver parity wording.
- node, HVAC, IdealLoads, plant, meter, sizing, autosizing, or ExampleFiles
  numerical conformance wording beyond already promoted v0.8/v0.9 cases.

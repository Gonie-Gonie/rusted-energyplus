# v0.15.0 Plan

Purpose: add the first manifest-backed plant-loop diagnostic baseline using a
small plant-only EnergyPlus fixture, while keeping plant work below numerical
conformance level.

This is not a plant, HVAC, node, meter, sizing, or ExampleFiles numerical
conformance claim.

## Evidence Targets

- `plant_loop_diagnostic_001` records a clean EnergyPlus oracle baseline for a
  PlantLoadProfile-based plant loop.
- `plant-state` and `plant-equipment` output classes separate plant evidence
  from general HVAC output requests.
- `plant-loop-diagnostic-smoke` validates the manifest, runs the oracle,
  checks zero EnergyPlus warnings, compiles the staged epJSON, verifies plant
  graph edges, and checks baseline-only report rows.
- `plant.md` and `output-variable-matrix.md` record the diagnostic case and
  non-claim boundary.
- `v0.15-verify` packages the case, smoke gate, docs, and promoted v0.8/v0.9
  numerical evidence pack.

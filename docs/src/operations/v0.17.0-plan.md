# v0.17.0 Plan

Purpose: define Case Manifest and Output Request Schema v2 before adding more
ExampleFiles coverage.

This milestone is schema and validation infrastructure only. It does not add a
new numerical conformance claim.

## Targets

- `case.toml` v2 schema with explicit case id, source kind, source file,
  oracle version, weather, tier, comparison class, and claim boundary.
- output request schema for variables and meters.
- case tier enum: Tier A, Tier B, Tier C.
- comparison class enum: `smoke`, `baseline-only`, `diagnostic-only`,
  `conformance`.
- output level enum: `required`, `optional`, `baseline`, `diagnostic`,
  `conformance`.
- tolerance rule schema with absolute and relative tolerances.
- waiver schema with reason, owner, expiry, and affected variables.
- manifest validation CLI or dev command.
- migration plan for existing v0.8 through diagnostic cases.

## Gate

```powershell
.\scripts\dev.cmd manifest-validate-all
```

The command may be introduced during this milestone. Until then, v0.17 remains
planned and not release-ready.

## Boundary

Conformance cases must require output requests, tolerance policy, report
artifacts, and a blocking gate. Baseline-only and diagnostic-only cases must
force `conformance_claim = false`.

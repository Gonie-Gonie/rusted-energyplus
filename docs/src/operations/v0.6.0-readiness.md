# v0.6.0 Readiness

Status: diagnostic-ready when `.\scripts\dev.cmd v0.6-verify` passes.

## Release Definition

v0.6.0 locks the first output/trace/report diagnostic infrastructure:

```text
ResultStore diagnostic + zone MAT diagnostic report + compare regression artifacts
```

Supported commands:

```powershell
eplus-rs run first-zone <input.epJSON> <weather.epw> [--hours N]
eplus-rs compare zone-temperature <input.epJSON> <weather.epw> <eplusout.eso> --report-dir <dir>
eplus-rs conformance diagnostic-report <case.toml> <oracle-root> <output-root>
.\scripts\dev.cmd compare-regression
```

## Completion Checklist

| Requirement | Evidence |
|---|---|
| ResultStore diagnostic path | `first-zone-smoke.ps1` verifies native zone-temperature and weather output series |
| zone MAT extraction | `compare-zone-smoke.ps1` reads EnergyPlus ESO, computes deltas, and writes diagnostic artifacts |
| manifest-driven diagnostic report | `conformance-diagnostic-report-smoke.ps1` runs `zone_temperature_diagnostic_001` and embeds manifest metadata |
| compare artifact suite | `compare-regression.ps1` writes `trace.json`, `compare-summary.json`, `compare-report.md`, and `profile-summary.json` |
| schema boundary | `zone_temperature_diagnostic_001` uses `comparison_class = "diagnostic-only"`, no tolerances, and `conformance_claim = false` |
| false-claim boundary | `strict-no-false-conformance` forbids zone-temperature pass wording and requires diagnostic-only markers |

## Verification Command

```powershell
.\scripts\dev.cmd v0.6-verify
```

## Boundary

v0.6.0 does not claim EnergyPlus-compatible heat balance, zone temperature,
surface temperature, HVAC, plant, meter, SQL, or whole-building simulation
conformance. A successful diagnostic extraction means artifacts were produced
and deltas were reported; it does not mean the deltas passed a tolerance.

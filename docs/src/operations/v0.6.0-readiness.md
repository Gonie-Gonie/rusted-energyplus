# v0.6.0 Readiness

Status: in progress.

## Release Definition

v0.6.0 introduces runtime plumbing diagnostics and the first native
`ResultStore` path:

```text
SimulationModel + EPW dry-bulb -> SimulationState -> ResultStore diagnostic
```

Supported command:

```powershell
eplus-rs run first-zone <input.epJSON> <weather.epw> [--hours N]
eplus-rs compare zone-temperature <input.epJSON> <weather.epw> <eplusout.eso>
```

## Current Subset

- first `Zone` only
- opaque `BuildingSurface:Detailed` areas
- first-layer `Construction -> Material` resistance
- `Material` thickness/conductivity and `Material:NoMass` thermal resistance
- `Schedule:Constant`
- `Schedule:Compact` all-days `Until` segment subset
- `OtherEquipment` as constant sensible internal gain
- `RunPeriod` date range parsed into an hourly time-axis foundation
- EPW outdoor dry-bulb as the weather driver, with additional weather fields
  parsed for future compatibility work
- zone geometry summary for surface count, floor area, derived volume, and
  exterior wall area
- hourly `ResultStore` series for zone mean air temperature and outdoor dry-bulb

## Verification Command

```powershell
.\scripts\first-zone-smoke.cmd
.\scripts\compare-zone-smoke.cmd
.\scripts\geometry-smoke.cmd
.\scripts\check.cmd
```

The smoke runs the EnergyPlus-generated oracle epJSON fixture with the locked
EnergyPlus weather file and verifies that the Rust runtime writes a structured
two-series `ResultStore`. The zone comparison smoke also runs a 24-hour
EnergyPlus oracle case, converts it to epJSON, reads `Zone Mean Air Temperature`
from ESO, and reports Rust-vs-EnergyPlus deltas from the same first-zone output
path. This is diagnostic extraction, not tolerance-based conformance.
The geometry smoke summarizes EnergyPlus-converted input geometry for future
EIO/internal-variable comparison, but it does not yet compare against an oracle
with tolerances.

## Boundary

This is a deterministic RC-style toy diagnostic, not full EnergyPlus heat
balance. Solar, windows, infiltration, HVAC, radiant exchange, warmup, sizing
periods, and exact zone-temperature comparison remain future milestones. The
output must not be described as EnergyPlus-compatible simulation evidence.

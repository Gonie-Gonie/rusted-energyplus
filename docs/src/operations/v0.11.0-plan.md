# v0.11.0 Plan

Purpose: start the air-side HVAC node milestone with baseline-only node output
evidence and a diagnostic Rust node-state projection on top of the v0.10
thermostat, equipment, IdealLoads, NodeList, and node registry foundation.

This is not a node or HVAC numerical conformance claim.

## Case

`air_side_node_diagnostic_001` is the first v0.11 case. It reuses the
one-zone IdealLoads shape from v0.10 but adds explicit `System Node` output
requests for every typed node in that fixture:

- `ZONE ONE INLET`
- `ZONE ONE AIR NODE`
- `ZONE ONE RETURN`

Requested node variables:

- `System Node Temperature`
- `System Node Humidity Ratio`
- `System Node Mass Flow Rate`

`System Node Setpoint Temperature` stays outside this first gate because the
inlet and return nodes emit the EnergyPlus `-999` sentinel in the current
fixture. It remains a v0.11+ follow-up variable.

## Evidence

The gate must prove:

- `comparison_class = "diagnostic-only"`
- `conformance_claim = false`
- `tolerance_policy: none`
- `status: baseline-only` for the EnergyPlus report skeleton
- `status: projected` and `algorithm_parity: false` for the Rust node-state
  projection
- zero EnergyPlus warnings, severes, and fatals
- 13 requested ESO series total
- 9 `node-state` ESO series total
- 9 Rust `ResultStore` node-state projection series total
- 24 samples for each requested node-state series
- nonzero samples for every requested node-state series
- typed node registry count remains 3
- one NodeList member graph edge
- one IdealLoads supply-node graph edge
- one zone air-node graph edge

## Gate

Required command:

```powershell
.\scripts\dev.cmd air-side-node-diagnostic-smoke
```

Release verification:

```powershell
.\scripts\dev.cmd v0.11-verify
```

## Deferred

v0.11 does not yet claim:

- node temperature, humidity, flow, or setpoint parity
- fan, coil, PTAC, CAV, or air-loop component parity
- sizing or autosizing parity
- availability-manager parity
- outdoor-air, economizer, ventilation, heat-recovery, or humidity-control parity
- meter or fuel accounting parity

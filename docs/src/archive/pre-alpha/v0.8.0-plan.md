# v0.8.0 Plan

Purpose: promote the first heat-balance result from diagnostic extraction to a
tolerance-gated conformance case.

## Promoted Case

`heat_balance_nomass_001` is the only v0.8 heat-balance conformance case.

It covers:

- one `Zone`
- six opaque `BuildingSurface:Detailed` surfaces
- `Material:NoMass`
- one `Construction`
- adiabatic outside boundary conditions
- no internal gains, windows, solar, infiltration, HVAC, plant, moisture, EMS,
  or sizing-period claim
- hourly `Zone Mean Air Temperature`

## Source Reference

The runtime initialization is aligned with EnergyPlus 26.1.0:

- `src/EnergyPlus/DataHeatBalance.hh`
- `DataHeatBalance::ZoneInitialTemp`
- `DataHeatBalance::SurfInitialTemp`

For v0.8, Rust heat-balance MAT traces use the EnergyPlus 23 C initial zone
temperature default before any timestep advancement.

## Tolerance Gate

The case manifest declares:

- `comparison_class = "conformance"`
- `conformance_claim = true`
- output variable: `Zone Mean Air Temperature`
- frequency: hourly
- source: ESO
- tolerance: `max_abs = 0.000001`, `max_rmse = 0.000001`
- blocking gate: `scripts/dev.cmd compare-heat-balance-conformance`

The gate writes:

- `.runtime/heat-balance-conformance/26.1.0/heat_balance_nomass_001/compare/compare-summary.json`
- `.runtime/heat-balance-conformance/26.1.0/heat_balance_nomass_001/compare/compare-report.md`

## Not Claimed

This is not a dynamic exterior heat-balance claim. The existing Golden weather
zone-temperature diagnostic remains diagnostic-only.

Not claimed in v0.8:

- outdoor surface heat-balance parity
- surface inside/outside face temperature conformance
- mass construction transient response
- solar, longwave radiation, wind, ground, or sky exchange parity
- infiltration, ventilation, HVAC, plant, or thermostat behavior
- warmup convergence parity beyond the declared initial temperature default

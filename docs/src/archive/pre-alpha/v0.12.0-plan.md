# v0.12.0 Plan

Purpose: harden the v0.11 air-side node diagnostic milestone by locking the
EnergyPlus source-function map for node registration, node state updates, and
node output registration before any node or IdealLoads numerical claim is
promoted. The existing Rust node-state projection remains diagnostic-only
plumbing evidence.

This is not a node, IdealLoads, HVAC, plant, or ExampleFiles numerical
conformance claim.

## Evidence Targets

- `node-state-source-map.md` identifies the EnergyPlus 26.1.0 source files,
  routines, data fields, and v0.11 fixture node paths.
- `output-variable-source-map.md` records the node-state variables against the
  new source-map entry.
- `hvac.md` points future IdealLoads/node work at the source map instead of
  leaving system-node behavior as an unmapped TODO.
- The v0.11 Rust node-state projection is documented as diagnostic-only,
  `NodeStateStore`-backed, `algorithm_parity: false` evidence, not a
  numerical claim.
- The Rust node-state projection artifact records the v0.12 source-map path,
  timestamp rule, warmup rule, and setpoint sentinel exclusion rule.
- `v0.12-verify` checks source files, source-map markers, docs, tests, and
  false-claim boundaries.

## Release Gate

```powershell
.\scripts\dev.cmd v0.12-verify
```

## Boundary

Allowed in v0.12:

- source-map documentation
- release-gate checks for the source-map entry
- diagnostic-only or baseline-only node report and projection improvements
- projection artifact policy fields for source map, timestamp, warmup, and
  sentinel handling
- no tolerance-gated node claim

Not allowed in v0.12:

- node temperature, humidity ratio, mass flow, or setpoint conformance wording
- IdealLoads load conformance wording
- HVAC, plant, air-loop, fan, coil, sizing, or meter conformance wording
- promotion of `air_side_node_diagnostic_001` beyond `diagnostic-only`

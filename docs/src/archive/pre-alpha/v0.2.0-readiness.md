# v0.2.0 Readiness

Status: ready when `.\scripts\dev.cmd v0.2-verify` passes.

## Release Definition

v0.2.0 locks the first conformance harness contract:

```text
case manifest + suite manifest + output requests + EnergyPlus baseline + baseline-only report skeleton
```

This milestone does not compare Rust simulation output against EnergyPlus.
`conformance_claim` remains `false` for the included fixture cases.

Supported commands:

```powershell
eplus-rs conformance validate-case <case.toml>
eplus-rs conformance baseline <case.toml> <oracle-root> <output-root>
eplus-rs conformance report-skeleton <case.toml> <baseline-case-dir> <report-root>
```

## Completion Checklist

| Requirement | Evidence |
|---|---|
| case metadata schema | `ep_conformance::ConformanceCase` parses and validates manifest fields |
| suite schema | `ep_conformance::ConformanceSuite` validates ordered case paths |
| output request schema | `OutputRequest` requires key, variable, frequency, class, and source artifact |
| false-claim boundary | schema rejects `conformance_claim = true` without conformance class, tolerances, report, and blocking gate |
| baseline generation | `conformance-baseline-smoke.ps1` stages IDF, converts epJSON, writes EnergyPlus artifacts, and writes `case-expanded.toml` |
| baseline-only reports | `conformance-report-smoke.ps1` writes `compare-report.md` and `compare-summary.json` from baseline outputs |
| fixture coverage | seed fixtures are schema-valid; later smoke manifests may be added to the suite without changing the v0.2 harness-only claim |

## Artifact Contract

Baseline generation must create a case directory under
`.runtime/conformance-baseline/<oracle-version>/<case-id>/` with at least:

- `input.idf`
- `input.epJSON`
- `eplusout.err`
- `eplusout.eso` for ESO-backed output requests
- `eplusout.eio` for EIO/static output requests when EnergyPlus writes it
- `case-expanded.toml`

Report skeleton generation must create a case directory under
`.runtime/conformance-report/<oracle-version>/<case-id>/` with:

- `compare-report.md`
- `compare-summary.json`

The expanded manifest and summary JSON must preserve each requested output's
`source` target: `eso`, `eio`, `mtr`, `sql`, or `csv`.

## Verification Command

```powershell
.\scripts\dev.cmd v0.2-verify
```

## Boundary

This milestone is harness-only. It does not claim Rust-vs-EnergyPlus numerical
agreement, schedule/weather parity, geometry parity, heat-balance behavior,
HVAC behavior, or plant behavior.
